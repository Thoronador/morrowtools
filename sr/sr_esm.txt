
Format of Skyrim's ESM PlugIn Files

(All that follows below is still an assumption!)


  Record
  ======
    4 bytes: char Name[4]
        4-byte record name string (not null-terminated)
    4 bytes: long Size
        Size of the record not including the 20 bytes of header data.
	16 bytes:
		Unknown values, probably four groups of 4-byte integers/ flags



  (Sub-)Record
  ============
    4 bytes: char Name[4]
        4-byte sub-record name string (not null-terminated)
    2 bytes: uint16 Size
        Size of the sub-record not including the 6 bytes of header data.
    ? bytes: Sub-Record data.
        Format depends on the sub-record type).


  Groups:
  =======
    GRUP
      4 byte - uint32 - size of group, including the header
      4 byte - char[4] - name of group to follow
      12 byte - unknown data
      ? bytes - group data


Records and their sub-records:

  TES4: Main header(?)
      HEDR (12 bytes)
        Unknown, possibly three 4-byte values?
        4 bytes - float, version number
        4 bytes - unknown
        4 bytes - unknown
      CNAM (string, NUL-terminated)
        author name
      INTV (4 bytes)
        unknown integer value


  GMST: Game Settings
      EDID: editor ID (string, 0-terminated)
      DATA: the data (string/int/float)
              format determined by first character of ID
              s = string (length varies)
              i = integer (4 bytes)
              f = floating point value (4 bytes)


  AACT: action(?)
      EDID (string, NUL-terminated)
          editorID


  AMMO: ammunition
      EDID (string, NUL-terminated)
          editorID
      OBND (12 bytes)
          unknown
      FULL (4 bytes, optional)
          string ID of object's full name
      MODL (string, NUL-terminated, optional)
          model path
      MODT (?? bytes, optional)
          unknown data
      YNAM (4 bytes)
          unknown, possibly form ID
      ZNAM (4 bytes)
          unknown, possibly form ID
      DESC (4 bytes)
          possibly string index
      KSIZ (4 bytes, optional)
          number of keywords in following KWDA
      KWDA (n * 4 bytes, optional)
          array of form IDs of keywords
      DATA (16 bytes)
          uint32_t  form ID of PROJ record
          uint32_t  flags(?)
          float     base damage
          uint32_t  value

          unknown data

          unknown data
      YNAM (4 bytes, optional)
          unknown data
      ZNAM (4 bytes, optional)
          unknown data

          unknown data
      KSIZ (4 bytes, optional)
          number of uint32_ts in KWDA
      KWDA (n * 4 bytes, optional)
          array of uint32_ts/form IDs
      DESC (4 bytes, optional)
          string ID of description (mostly zero, i.e. unused)
      MODL (4 bytes, can occur 1+ times)
          unknown data
      DATA (8 bytes)
          unknown data
      DNAM (4 bytes)
          unknown data

          unknown data


  DUAL: dual cast data
      EDID (string, NUL-terminated)
          editorID
      OBND (12 bytes)
          unknown data
      DATA (24 bytes)
          the dual cast data
            uint32_t  projectile form ID
            uint32_t  explosion form ID
            uint32_t  effect shader form ID
            uint32_t  hit effect art form ID (art object)
            uint32_t  impact data set form ID
            uint32_t  flags
                        0x01 - hit effect art inherits scale
                        0x02 - projectile inherits scale
                        0x04 - explosion inherits scale


  ECZN: encounter zone

  MUSC: music type
      EDID (string, NUL-terminated)
          editorID
      FNAM (4 bytes)
          uint32_t flags
              0x01 - Plays One Selection
              0x02 - Abrupt Transition
              0x04 - Cycle Tracks
              0x08 - Maintain Track Order (only allowed, if "Cycle Tracks" is set)
              0x20 - Ducks Current Track
      PNAM (4 bytes)
          uint16_t priority
          uint16_t Ducking dB (x100, i.e. 126 means 1.26)
                   allowed max.: 100.00 i.e. 10000; allowed min.: 0
      WNAM (4 bytes)
          float Fade Duration
      TNAM (n*4 bytes)
          Music track form IDs (record type MUST)


  MUST: music track
      EDID (string, NUL-terminated)
          editorID
      CNAM (4 bytes)
          type identifier
            0x6ED7E048 - single track (48 E0 D7 6E)
            0xA1A9C4D5 - silent track (D5 C4 A9 A1)
            0x23F678C3 - palette track (C3 78 F6 23)
      FLTV (4 bytes, optional)
          duration in seconds (only for silent tracks or palette tracks)
      DNAM (4 bytes, optional)
          float   Fade-Out (seconds) (allowed min.: 0; max.: 20s)
      SNAM (n* 4 bytes, optional)
          std::vector<uint32_t> form IDs of tracks (only in pallette typed tracks)
           ---> uint32_ts of value zero are used as layer seperator
      ANAM (string, NUL-terminated, optional)
          unknown music file path, usually .wav (does not occur for silent tracks)
      BNAM (string, NUL-terminated, optional)
          finale file path, usually .wav (does not occur for silent tracks)
      FNAM (n * 4 bytes, optional)
          std::vector<float>   Cue Points (in seconds)
      LNAM (12 bytes, optional)
          loop data
          float     Loop Begins (seconds)
          float     Loop Ends (seconds)
          uint32_t  Loop Count
      CITC (4 bytes, optional)
          condition count
      CTDA (32 bytes, can occur multiple times, optional)
          condition data
